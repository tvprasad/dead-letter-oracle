# ADR-001: Why Dead Letter Oracle

## Status
Accepted

## Context
We explored multiple ideas for the hackathon, ranging from highly ambitious system-level concepts to tighter product-oriented concepts. We needed a concept that balanced:
- originality
- feasibility in 2 days
- demo clarity
- MCP alignment
- real-world usefulness

## Decision
Build **Dead Letter Oracle — Live Incident Agent** as the primary submission.

## Rationale
Dead Letter Oracle was selected because:
- the problem is concrete and real
- the end-to-end flow is self-contained and demonstrable
- MCP tool orchestration is natural
- governance and explainability fit cleanly
- scope can be tightly controlled

## Consequences
The system is optimized for a tight, complete flow rather than maximum breadth or integration count.
